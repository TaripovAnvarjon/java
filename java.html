⚠ Bu qo'llanma JDK 8 uchun yozilgan. Yangi versiyalar uchun dev.java saytiga qarang. Java SE 9 va undan keyingi o'zgarishlar uchun Java Language Changes sahifasini ko'ring.

Dars: Ob'ektga Yo'naltirilgan Dasturlash Tushunchalari

Agar siz ilgari ob'ektga yo'naltirilgan dasturlash tilidan foydalanmagan bo'lsangiz, kod yozishni boshlashdan oldin bir necha asosiy tushunchalarni o'rganishingiz kerak bo'ladi. Bu dars sizga ob'ektlar, klasslar, meros, interfeyslar va paketlar haqida ma'lumot beradi. Har bir tushuncha haqiqiy hayot bilan bog'liq holda tushuntiriladi, shu bilan birga Java dasturlash tilining sintaksisiga kirish ham beriladi.

Ob'ekt nima?

Ob'ekt — o'zaro bog'liq holat va xatti-harakatdan iborat dasturiy to'plam. Dasturiy ob'ektlar ko'pincha kundalik hayotdagi real narsalarni modellash uchun ishlatiladi. Bu bo'lim ob'ekt ichida holat va xatti-harakat qanday saqlanishini, ma'lumotlarni yashirish (data encapsulation) tushunchasini va bunday yondashuv afzalliklarini tushuntiradi.

Klass nima?

Klass — ob'ektlar yaratish uchun ishlatiladigan qolip yoki shablon. Bu bo'lim real hayotdagi ob'ektning holati va xatti-harakatini modellaydigan klassni ta'riflaydi. Hatto oddiy klass ham holat va xatti-harakatni qanday aniq ifodalashini ko'rsatadi.

Meros nima?

Meros — dasturiy ta'minotni tartibga solish va tuzish uchun kuchli va tabiiy mexanizm. Bu bo'lim klasslar o'zlarining yuqori klasslaridan holat va xatti-harakatni qanday meros qilib olishini va Java tilining oddiy sintaksisi yordamida bir klassdan boshqa klass qanday hosil qilinishini tushuntiradi.

Interfeys nima?

Interfeys — klass va tashqi dunyo o'rtasidagi shartnoma. Klass interfeysni amalga oshirganda, u o'sha interfeys e'lon qilgan xatti-harakatlarni bajarishga va'da beradi. Bu bo'lim oddiy interfeysni ta'riflaydi va uni amalga oshiradigan klass uchun qanday o'zgarishlar kerakligini tushuntiradi.

Paket nima?

Paket — klasslar va interfeyslarni mantiqiy tarzda tartibga solish uchun ishlatiladigan nom maydoni. Kodni paketlarga joylashtirganda katta dasturiy loyihalarni boshqarish osonlashadi. Bu bo'lim nima uchun foydali ekanligini tushuntiradi va Java platformasi taqdim etadigan API bilan tanishtiradi.

Savol va Mashqlar

Bu bo'limdagi savol va mashqlardan foydalanib, ob'ektlar, klasslar, meros, interfeyslar va paketlar haqidagi tushunchalaringizni sinab ko'ring.

Ob'ekt nima?

Ob'ektlar ob'ektga yo'naltirilgan texnologiyani tushunishning kaliti hisoblanadi. Atrofingizga qarang — real hayotdagi ob'ektlarning ko'plab misollarini topasiz: itingiz, stolingiz, televizoringiz, velosipedingiz. Real hayotdagi ob'ektlarda ikkita umumiy xususiyat bor: ularda holat va xatti-harakat mavjud.

Misol — It:
Holati: ismi, rangi, zoti, ochmi yoki to'qmi
Xatti-harakati: hurish, narsani olib kelish, dum chayqash
Misol — Velosiped:
Holati: joriy tezlik, joriy panjara raqami, joriy aylanma tezlik
Xatti-harakati: panjara almashtirish, aylanma tezlikni o'zgartirish, tormoz bosish

Haqiqiy hayotdagi ob'ektni aniqlash ob'ektga yo'naltirilgan dasturlash tarzida fikr yuritishni boshlashning ajoyib usuli. Hozir atrofingizga qarang va ko'rgan har bir ob'ekt uchun o'zingizdan ikki savol bering: "Bu ob'ekt qanday holatlarda bo'lishi mumkin?" va "Bu ob'ekt qanday xatti-harakatni bajara oladi?"

Siz shunda real hayotdagi ob'ektlar murakkablik jihatidan farqlanishini ko'rasiz: stol chirog'ingizda faqat ikkita holat (yoniq, o'chiq) va ikkita xatti-harakat bo'lishi mumkin, radiola esa ko'proq holatlar (ovoz balandligi, joriy stansiya) va xatti-harakatlarni o'z ichiga oladi.

Dasturiy ob'ektlar

Dasturiy ob'ektlar ham kontseptual jihatdan real hayotdagi ob'ektlarga o'xshaydi: ularda ham holat va bog'liq xatti-harakat mavjud. Ob'ekt o'z holatini maydonlarda (fields), xatti-harakatini esa metodlarda (methods) saqlaydi. Metodlar ob'ektning ichki holatini boshqaradi va ob'ektlar o'rtasidagi muloqotning asosiy mexanizmi bo'lib xizmat qiladi.

Ma'lumotlarni yashirish (Data Encapsulation)

Ichki holatni yashirish va barcha muloqotni faqat ob'ektning metodlari orqali amalga oshirishni talab qilish — ma'lumotlarni yashirish (data encapsulation) deb ataladi. Bu ob'ektga yo'naltirilgan dasturlashning asosiy tamoyillaridan biridir.

Misol uchun, velosipedda faqat 6 ta panjara bo'lsa, panjarani o'zgartiruvchi metod 1 dan kichik yoki 6 dan katta qiymatni qabul qilishdan bosh tortishi mumkin. Bu tashqaridan keluvchi noto'g'ri qiymatlardan himoya qiladi.

💡 Afzalliklari:
• Modullilik — ob'ektning kodi boshqalardan mustaqil yoziladi va saqlanadi.
• Ma'lumotlarni yashirish — faqat metodlar orqali murojaat, ichki holat himoyalangan.
• Qayta foydalanish — ob'ektni dasturning boshqa qismlarida ishlatsa bo'ladi.
• Sozlash qulayligi — muammo topilsa, ob'ektni alohida tuzatish mumkin.

Klass nima?

Haqiqiy hayotda ko'pincha bir xil turdagi ko'plab alohida ob'ektlar uchraydi. Masalan, dunyoda minglab velosipedlar mavjud — hammasi bir xil chizma asosida qurilgan va bir xil qismlardan iborat. Ob'ektga yo'naltirilgan dasturlashda biz sizning velosipedingi velosipedlar klassining bir namunasi (instance) deb aytamiz.

Klass — ob'ektlar yaratish uchun ishlatiladigan qolip yoki shablon. Quyidagi klass velosipedni modellaydi:

class Bicycle {

    // Velosipedning holati
    int cadence = 0;
    int speed = 0;
    int gear = 1;

    // Velosipedning xatti-harakati
    void changeCadence(int newValue) {
         cadence = newValue;
    }

    void changeGear(int newValue) {
         gear = newValue;
    }

    void speedUp(int increment) {
         speed = speed + increment;
    }

    void applyBrakes(int decrement) {
         speed = speed - decrement;
    }

    void printStates() {
         System.out.println("cadence:" + cadence +
             " speed:" + speed +
             " gear:" + gear);
    }
}

Java tilining sintaksisi sizga yangi tuyulishi mumkin, lekin bu klassning dizayni velosiped ob'ektlari haqidagi oldingi muhokamaga asoslanadi. cadence, speed va gear maydonlari ob'ektning holatini, changeCadence, changeGear, speedUp va boshqa metodlar esa tashqi dunyo bilan muloqotini ifodalaydi.

Bicycle klassida main metodi yo'q — bu to'liq dastur emas, faqat velosiped ob'ektlarini yaratish uchun qolip. Ob'ektlarni yaratish va ulardan foydalanish vazifasi boshqa klassga yuklatilgan.

Quyidagi BicycleDemo klassi ikkita alohida Bicycle ob'ektini yaratadi va ularning metodlarini chaqiradi:

class BicycleDemo {
    public static void main(String[] args) {

        // Ikkita Bicycle ob'ekti yaratish
        Bicycle bike1 = new Bicycle();
        Bicycle bike2 = new Bicycle();

        // bike1 ni boshqarish
        bike1.changeCadence(50);
        bike1.speedUp(10);
        bike1.changeGear(2);
        bike1.printStates();

        // bike2 ni boshqarish
        bike2.changeCadence(50);
        bike2.speedUp(10);
        bike2.changeGear(2);
        bike2.changeCadence(40);
        bike2.speedUp(10);
        bike2.changeGear(3);
        bike2.printStates();
    }
}

Ushbu sinov natijasi ikki ob'ektning bir-biridan mustaqil holatga ega ekanini ko'rsatadi:

cadence:50  speed:10  gear:2
cadence:40  speed:20  gear:3

Meros (Inheritance) nima?

Har xil turdagi ob'ektlar ba'zida bir-biriga ma'lum miqdorda o'xshash bo'ladi. Tog' velosipedlari, yo'l velosipedlari va tandem velosipedlari, masalan, hammasi velosipedning umumiy xususiyatlarini (joriy tezlik, joriy aylanma tezlik, joriy panjara raqami) baham ko'radi. Lekin har birida farqli xususiyatlar ham bor: tandem velosipedida ikki o'rindiq va ikki rul bor; yo'l velosipedida egilgan rul; ba'zi tog' velosipedlarida past peredachani ta'minlovchi qo'shimcha tishli halqa mavjud.

Ob'ektga yo'naltirilgan dasturlash klasslarning umumiy holat va xatti-harakatni boshqa klasslardan meros qilib olishiga imkon beradi. Ushbu misolda Bicycle endi MountainBike, RoadBike va TandemBike klasslarining yuqori klassi (superclass) bo'ladi.

class MountainBike extends Bicycle {

    // MountainBike'ga xos yangi maydon va metodlar
    // shu yerga yoziladi

}

Bu MountainBike ga Bicycle ning barcha maydon va metodlarini beradi, shu bilan birga o'ziga xos xususiyatlarini alohida ifodalash imkonini qoldiradi. Bu quyi klass kodini o'qishni osonlashtiradi. Biroq har bir yuqori klass ta'riflaydigan holat va xatti-harakatni to'g'ri hujjatlashtirish muhim — chunki bu kod quyi klassning manba faylida ko'rinmaydi.

💡 Java tilida meros qoidasi:
Har bir klass faqat bitta bevosita yuqori klassga ega bo'la oladi — bu yagona meros (single inheritance) deyiladi. Istisno faqat Object klassi: uning yuqori klassi yo'q. Har bir yuqori klassning esa cheksiz miqdorda quyi klassi bo'lishi mumkin.

Meros zanjiri

Klasslar klasslardan, ular boshqa klasslardan hosil bo'lishi mumkin — va hokazo, oxir-oqibat eng yuqori klass — Object gacha. Bunday klass meros zanjiri bo'ylab Object gacha bo'lgan barcha klasslarning avlodi deb hisoblanadi.

Interfeys (Interface) nima?

Siz allaqachon bilganingizdek, ob'ektlar o'zlarining ochiq metodlari orqali tashqi dunyo bilan muloqot qiladi. Metodlar ob'ektning tashqi dunyo bilan interfeysini shakllantiradi. Masalan, televizoringizning old tomonidagi tugmalar — siz bilan uning ichidagi elektr sxema orasidagi interfeys. Siz televisorni yoqish yoki o'chirish uchun "power" tugmasini bosasiz.

Java tilida interfeys — bu klass va tashqi dunyo o'rtasidagi shartnoma (contract). Klass interfeysni amalga oshirganda (implements), u o'sha interfeys e'lon qilgan barcha xatti-harakatlarni bajarishga va'da beradi. Bu shartnomani kompilyator qurilish vaqtida nazorat qiladi.

Velosiped uchun interfeys quyidagicha ko'rinishi mumkin:

interface Bicycle {

    //  aylanma tezlikni qo'shish (rpm)
    void changeCadence(int newValue);

    void changeGear(int newValue);

    void speedUp(int increment);

    void applyBrakes(int decrement);
}

Bu interfeysni amalga oshirish uchun klassning nomi o'zgartiriladi — ACMEBicycle deb nomlangan, va implements Bicycle qo'shiladi:

class ACMEBicycle implements Bicycle {

    int cadence = 0;
    int speed = 0;
    int gear = 1;

    // Kompilyator endi changeCadence, changeGear,
    // speedUp va applyBrakes metodlarining barchasi
    // mavjudligini tekshiradi. Agar birontasi
    // yetishmasa — kompilyatsiya muvaffaqiyatsiz tugaydi.

    public void changeCadence(int newValue) {
         cadence = newValue;
    }

    public void changeGear(int newValue) {
         gear = newValue;
    }

    public void speedUp(int increment) {
         speed = speed + increment;
    }

    public void applyBrakes(int decrement) {
         speed = speed - decrement;
    }

    public void printStates() {
         System.out.println("cadence:" + cadence +
             " speed:" + speed +
             " gear:" + gear);
    }
}
💡 Eslatma: ACMEBicycle klassini kompilyatsiya qilish uchun amalga oshirilgan interfeys metodlarining boshiga public kalit so'zini qo'shish kerak. Sababi keyinchalik "Klasslar va Ob'ektlar" va "Interfeyslar va Meros" bo'limlarida tushuntiriladi.

Interfeysni amalga oshirish klass o'z xatti-harakati haqida rasmiyroq bo'lib qolishiga imkon beradi. Agar klass interfeysni amalga oshirishini e'lon qilsa, interfeysdagi barcha metodlar klass manba faylida mavjud bo'lishi shart — aks holda kompilyatsiya muvaffaqiyatsiz tugaydi.

Paket (Package) nima?

Paket — o'zaro bog'liq klasslar va interfeyslar to'plamini tartibga soluvchi nom maydoni. Paketlarni kompyuteringizdagi turli papkalar kabi tasavvur qilish mumkin: HTML sahifalarni bir papkada, rasmlarni boshqa papkada, dastur va skriptlarni yana boshqa papkada saqlashingiz mumkin.

Java dasturlash tilida yozilgan dasturiy ta'minot yuzlab yoki minglab alohida klasslardan iborat bo'lishi mumkin. Shuning uchun bog'liq klasslar va interfeyslarni paketlarga joylashtirish orqali tartibda saqlash maqsadga muvofiq.

Java Platformasining API'si

Java platformasi o'z dasturlaringizda foydalanish uchun mos bo'lgan ulkan sinf kutubxonasini (paketlar to'plami) taqdim etadi. Ushbu kutubxona Application Programming Interface (API) — ya'ni Dasturlash Interfeysi deb ataladi. Uning paketlari umumiy dasturlash vazifalarini ifodalaydi:

API'dan misollar:
• String ob'ekti — matn satrlari holati va xatti-harakatini o'z ichiga oladi.
• File ob'ekti — dasturga fayl tizimidagi faylni yaratish, o'chirish, solishtirish yoki o'zgartirishga imkon beradi.
• Socket ob'ekti — tarmoq rozetkasini yaratish va ishlatishni ta'minlaydi.
• Turli GUI ob'ektlari — tugmalar, tekisliklar va boshqa grafik interfeys elementlarini boshqaradi.

Haqiqiy dasturlash vazifalaringizni amalga oshirishda mavjud Java platformasi kodlari bilan o'zingizning kodlaringizni birgalikda ishlatishingiz mumkin.

💡 API hujjatlari:
Java Platform API Spetsifikatsiyasi — barcha paketlar, interfeyslar, klasslar, maydonlar va metodlarning to'liq ro'yxatini o'z ichiga oladi. Dasturchi sifatida bu sahifa siz uchun eng muhim ma'lumot manbaiga aylanadi — uni brauzeringizga xatchoʻp qilib qo'ying.

Savol va Mashqlar: OOP Tushunchalari

Savollar

1. Haqiqiy hayotdagi ob'ektlar dasturiy ob'ektlarga o'xshashligi nimadan iborat?
Ikkalasi ham holat va xatti-harakatga ega. Dasturiy ob'ekt holatini maydonlarda (fields), xatti-harakatini esa metodlarda (methods) saqlaydi.
2. Holat va xatti-harakat o'rtasidagi farq nima? Ikkitadan misol keltiring.
Holat — ob'ektning hozirgi qiymatlari. Misol: it — ismi, rangi; velosiped — joriy tezligi.
Xatti-harakat — ob'ekt bajara oladigan amallar. Misol: it — hurish; velosiped — tormoz bosish.
3. Stol chirog'ini dasturiy ob'ekt sifatida ifodalang. Uning holati va xatti-harakatini aniqlang.
Holati: yoniq yoki o'chiq, yorug'lik darajasi (agar sozlanishi mumkin bo'lsa).
Xatti-harakati: yoqish, o'chirish, yorug'likni oshirish yoki kamaytirish.
4. Klass nima?
Klass — ob'ektlar yaratish uchun ishlatiladigan qolip yoki shablon. U ob'ektning holatini (maydonlar) va xatti-harakatini (metodlar) ta'riflaydi.
5. Haqiqiy hayotdagi ob'ektlarni (masalan, it yoki velosiped) modellashda klasslar qanday foydali?
Klasslar haqiqiy narsalarning holati va xatti-harakatini dastur ichida aniq ko'rinishda ifodalash imkonini beradi. Bir klassdan ko'plab mustaqil ob'ektlar yaratilishi mumkin.
6. Meros (inheritance) nima?
Meros — bir klassning boshqa klassdan uning barcha maydon va metodlarini qabul qilishi. extends kalit so'zi ishlatiladi. Java tilida yagona meros qo'llaniladi.
7. Interfeys (interface) nima?
Interfeys — klass va tashqi dunyo o'rtasidagi shartnoma. Klass interfeysni amalga oshirganda, interfeysdagi barcha metodlarni bajarishga va'da beradi; bu shartnomani kompilyator nazorat qiladi.
8. Paket (package) nima?
Paket — klasslar va interfeyslarni mantiqiy tarzda tartibga solish uchun ishlatiladigan nom maydoni. Katta loyihalarda kodni boshqarishni osonlashtiradi.

Mashqlar

Mashq 1
Ushbu darsning boshida kuzatgan real hayotdagi ob'ektlaringiz uchun yangi klasslar yarating. Sintaksisni eslay olmasangiz, Bicycle klassiga qarang.
Mashq 2
Yuqorida yaratgan har bir yangi klass uchun uning xatti-harakatini ta'riflovchi interfeys yarating, so'ngra klassni shu interfeysni amalga oshirishga (implements) majbur qiling. Bir yoki ikkita metodni yozmay qoldiring va kompilyatsiya qilishga urinib ko'ring — xato xabari qanday ko'rinadi?
